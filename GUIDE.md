# 가이드 문서

## API 소개
0. ROOT 폴더에 있는 apiTest.http 파일 참조
1. 회원가입 및 로그인
    - 회원가입 등록 API : /users (POST)
    - 로그인 API : /login (POST)
2. 이용자 API
    - User 목록 불러오기 API : /users (GET) - 인증 필요
    - User 본인 정보 불러오기 API : /users/{id} (GET) - 인증 필요: 키가 본인일 때만
    - User 생성 API : /users (POST)
    - User 업데이트 API : /users/{id} (PUT) - 인증 필요: 키가 본인일 때만
    - User 탈퇴 API : /users/{id} (DELETE) - 인증 필요: 키가 본인일 때만
3. 게시판 API
    - 게시판 목록 불러오기 API : /posts (GET)
    - 선택된 게시판 글 확인 API : /posts/{id} (GET)
    - 게시판 생성 API : /posts (POST) - 인증 필요
    - 게시판 글 업데이트 API : /posts/{id} (PUT) - 인증 필요: 키가 본인일 때만
    - 게시판 글 삭제 API : /posts/{id} (DELETE) - 인증 필요: 키가 본인일 때만

## DATABASE 설치 가이드
1. ROOT 폴더 경로에서 아래 명령어 입력
```
docker-compose up
```
2. 아래 계정 정보로 접속
```
url: jdbc:postgresql://localhost:5432/postgres
id: user
password: 1234
```
3. 아래 쿼리로 테이블 생성
```
create table tb_user
(
    id         bigint generated by default as identity
        primary key,
    created_at timestamp(6) not null,
    deleted_at timestamp(6),
    updated_at timestamp(6),
    age        integer,
    gender     varchar(255),
    name       varchar(255) not null,
    password   varchar(255) not null,
    phone      varchar(255),
    username   varchar(255) not null
);

create table tb_post
(
    id         bigint generated by default as identity
        primary key,
    created_at timestamp(6) not null,
    deleted_at timestamp(6),
    updated_at timestamp(6),
    content    oid          not null,
    title      varchar(255) not null,
    user_id    bigint       not null
        constraint fkhx7a7k3pf66vpddqg5pr12anw
            references tb_user
);
```
4. 종료시 아래 명령어 입력
```
docker-compose down
```
